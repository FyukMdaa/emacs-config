#+title: emacs-config

[[https://github.com/emacs-twist/twist.nix][twist.nix]] で管理する Emacs 設定の雛形です。

- init.el / early-init.el は =init.org= / =early-init.org= という
  org ファイルで書き、Nix (=emacs-twist/org-babel=) が tangle します。
- Nix 環境さえあれば =nix run= だけでどこでも同じ Emacs が使えます。
- nixos-config に flake input として登録し、home-manager 経由でパッケー
  ジとして組み込めます。
- パッケージ管理は =use-package= ではなく [[https://codeberg.org/pkal/setup.el][setup.el]] を使います。

パッケージは雛形として最小限 (=which-key= のみ) に留めてあります。

* ディレクトリ構成

#+begin_example
.
├── flake.nix          設定の本体。twist.lib.makeEnv でビルドする
├── home-module.nix    nixos-config / home-manager 用モジュール
├── early-init.org     early-init.el の literate source
├── init.org           init.el の literate source
├── nix/
│   └── registries.nix MELPA / GNU ELPA / NonGNU ELPA / Emacsmirror
├── recipes/           registries.nix にないパッケージ用の自作レシピ
└── lock/              twist.nix が生成するロックファイル置き場
    ├── flake.lock       (nix run .#lock で生成)
    ├── archive.lock
    └── flake.nix
#+end_example

* 使い方

** 前提

Nix (flakes 有効化済み) が入っていること。 [[https://nixos.org/download][公式サイト]] や
[[https://github.com/DeterminateSystems/nix-installer][Determinate Nix Installer]] などでインストールしてください。

** 最初の一回だけ: ロックファイルを生成する

このリポジトリを clone した直後は =lock/= が空です。ビルドする前に一度、
ロックファイルを生成してください。

#+begin_src sh
nix run .#lock
#+end_src

(=lock= は =flake.nix= / =archive.lock= を生成したあと、内部で
=nix flake lock= も実行して =flake.lock= まで一気に更新します。
ELPA アーカイブ由来のパッケージだけを更新したいときは、代わりに
=nix run .#update= を使ってください。)

生成された =lock/= 以下のファイルはすべて Git にコミットしてください。
これでパッケージのバージョンが完全に固定され、他の環境でも同じ
Emacs をそのまま再現できます。

** そのまま起動する (nix run)

Nix さえ入っていれば、インストールせずにこの設定の Emacs をどこでも
起動できます。

#+begin_src sh
# リポジトリを clone している場合
nix run .

# clone せず GitHub から直接 (公開リポジトリにした場合)
nix run github:your-name/emacs-config
#+end_src

=nix build .= すれば =./result/bin/emacs= としてバイナリも取り出せます。

** nixos-config / home-manager に組み込む

flake input として登録し、=home-module.nix= を import します。

#+begin_src nix
{
  inputs = {
    # ...
    emacs-config.url = "github:your-name/emacs-config";
  };

  outputs = { self, nixpkgs, home-manager, emacs-config, ... }: {
    homeConfigurations.you = home-manager.lib.homeManagerConfiguration {
      # ...
      modules = [
        emacs-config.homeModules.default
        {
          programs.emacs-twist.enable = true;
        }
      ];
    };
  };
}
#+end_src

`environment.systemPackages` に直接パッケージとして足すだけでも構いま
せん。

#+begin_src nix
environment.systemPackages = [
  emacs-config.packages.${pkgs.system}.default
];
#+end_src

* 設定を編集する

** init.org / early-init.org を書き換える

普通の org ファイルとして、見出しと =#+begin_src emacs-lisp= ブロックで
書きます。ビルド時に Nix が tangle するので、Emacs を起動して
=org-babel-tangle= を手で実行する必要はありません
(手元で試しに tangle して読みたいときは =C-c C-v C-t= で構いません。
生成される =init.el= / =early-init.el= は =.gitignore= 済みです)。

** パッケージを追加する

setup.el の =:package= を使います。 =init.org= の tangle 時に
=(setup (:package foo) ...)= の =foo= が自動的に検出され、
=nix/registries.nix= に列挙したレジストリ (MELPA → GNU ELPA →
NonGNU ELPA → Emacsmirror の順) から探してビルドされます。

#+begin_src emacs-lisp
(setup (:package foo)
  (:require foo)
  (foo-mode 1))
#+end_src

どのレジストリにもないパッケージだけ、=recipes/= に MELPA 形式の
レシピを追加してください (=recipes/README.md= 参照)。

パッケージを追加・変更したら、ロックファイルを更新するのを忘れずに:

#+begin_src sh
nix run .#lock
#+end_src

** setup.el について

=use-package= の代わりに [[https://codeberg.org/pkal/setup.el][setup.el]] を使っています。 setup.el 標準の
=:package= (中身は本来 =package-install= を呼ぶだけのもの) は、
twist.nix がすでに Nix store 上にパッケージをビルド・配置しているため
不要です。そこで =init.org= の冒頭で =:package= を no-op に上書きして
います。

setup.el が提供するキーワードの全体像は =M-x describe-function RET
setup RET= や upstream ([[https://codeberg.org/pkal/setup.el]],
[[https://www.emacswiki.org/emacs/SetupEl]]) を参照してください。

* 補足

- =init.org= / =early-init.org= は実際に (Nix 上ではなく) 通常の
  =org-babel-tangle= で tangle し、生成された =init.el= /
  =early-init.el= を実際の Emacs (29.3) 上で読み込んで、setup.el 本体
  (=:package= の no-op 化・=:require=・=:hook= の展開結果を含む) が
  期待通りに動くことを確認済みです。
- =flake.nix= / =nix/registries.nix= / =home-module.nix= は
  =nix-instantiate --parse= で構文を確認し、さらに
  =twist.lib= (=makeEnv= / =parseSetup= / =makeApps=) や
  home-manager モジュール (=modules/home-manager.nix=) の実際のソースを
  読んで、引数名やデフォルト値 (=parseSetup= のデフォルトキーワードが
  =":package"= であること、=makeApps= が生成する app 名が
  =lock= / =update= であることなど) を突き合わせています。
- ただし、パッケージのビルドそのもの (=nix build= / =nix run=) は
  この環境のネットワーク制限のためフルには実行できていません。
  初回は必ず =nix run .#lock= を実行してロックファイルを生成し、
  そのまま =nix run .= が通るか確認してください。動かない場合は
  =nix flake show= や twist.nix 本体のソース
  ([[https://github.com/emacs-twist/twist.nix]]) を参照してください。
